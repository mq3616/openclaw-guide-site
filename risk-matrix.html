改动风险矩阵 ⚠️

不是所有配置改动风险都一样。这页的作用，就是让你一眼知道哪些能放心试，哪些一定要小心。

🟢 低风险改动

  • 改某个 bot 的 USER.md 回复偏好
  • 改某个 bot 的 SOUL.md 语气描述
  • 补充 TOOLS.md 里的本地说明
  • 给教程站补页面 😄

特点：通常只影响一个 bot 的表现，不容易把系统整体搞挂。

🟡 中风险改动

  • 改某个 agent 的 workspace
  • 改某个 bot 的 thinkingDefault
  • 给某个 agent 增加或收紧 skills
  • 改 Discord 的 groupPolicy / allowlist

特点：通常不会让网关直接挂，但很容易让 bot 行为明显变化。

🔴 高风险改动

  • 改 bindings[]，尤其是多 bot 账号路由
  • 改 channels.discord.accounts.*.token
  • 改 gateway.auth / gateway.bind / gateway.port
  • 大范围重写多个 bot 的 workspace
  • 同时改路由层 + 技能层 + 人格层

特点：一旦改错，常见后果是 bot 不回、回错人格、无法接入、整条入口失效。

🧠 一条非常实用的规则

越靠近 token / binding / gateway，风险越高。越靠近 USER / SOUL / TOOLS，风险越低。
