Bot、Agent、Binding 三者关系

这是多 bot 配置最核心的概念。如果把它搞清楚，你以后看配置会快很多。

Bot 账号

是 Discord 上真实存在的机器人账号，对应 channels.discord.accounts.<accountId>。

Agent

是处理消息的“脑子”，对应 agents.list[].id，有自己的 workspace、memory、人格和行为规则。

Binding

是路由规则，把某个 channel/accountId/peer 的消息送到某个 agent。

bindings[] 里每个 key 的作用

agentId命中这条规则后，把消息交给哪个 agent。
match.channel匹配哪个渠道，比如 discord / telegram / whatsapp。
match.accountId匹配哪个账号实例。多 bot 最常用的就是这一层。
match.peer更细粒度匹配，比如指定某个群、某个用户、某个 thread。你当前主要还没用到这一层细分。
你现在采用的是非常清晰的一种结构：一个 Discord bot 账号，对应一个 agent。这比“多个 bot 共用一个 agent”更容易做角色分工。

多 bot 设计里最容易踩的坑

  • 把 bot 和 agent 当成同一个概念。 实际上 bot 是外部账号，agent 是内部脑子。
  • 只建多个 bot，不拆 workspace。 这样表面多个机器人，实则还是一个人格。
  • 绑错 accountId。 会出现 A 机器人账号说的像 B。
  • 以为不同 bot 自动就有不同 skills。 不会，skill 是否不同要看 agent 配置和 skill allowlist。
