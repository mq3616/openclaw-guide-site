Discord 配置重点，只挑常用的讲 💬

OpenClaw 的 Discord 配置很多，这里只讲你最常动、最常踩坑、最影响 bot 使用体验的几块。尤其是多 bot、多账号、多 agent 场景下，哪些 key 你一定要看，哪些不必天天碰。

📦 channels.discord 这一层最常用的 key

enabled是否启用 Discord 渠道。关掉后，这台 OpenClaw 就不再处理 Discord。
defaultAccount默认 Discord 账号 id。你这里是 default。如果某些地方没有明确指定 accountId，就会先看它。
groupPolicy群聊默认策略。常见值：allowlist / open / disabled。你现在是 open。
直觉理解：它决定 bot 在群环境里默认是收着、放开、还是彻底不理。
streaming.mode是否边生成边回。你现在是 off。
直觉理解：开了以后更像实时打字，关了更像一次成稿。
accounts多 Discord bot 账号列表。多 bot 配置最核心就在这里。
一句话：这一层定义“你接了哪些 Discord 机器人号”。

🤖 channels.discord.accounts.<id> 怎么理解

default默认 bot 账号，通常和 main agent 搭配。
trading_report_bot专用 Discord bot 账号，通过 binding 路由到 trading_report_bot agent。
family_health_bot同理，路由到 family_health_bot。
library_growth_bot路由到 library_growth_bot。
info_collection_bot路由到 info_collection_bot。
skills_master_bot路由到 skills_master_bot。
token这个账号对应的 Discord bot token。非常敏感，只能本机保管。
注意：token 只是“登录凭据”，不会定义 bot 性格。

🧠 最重要的设计原则

  1. 一个 bot 账号不等于一个人格。 真正的人格还是 agent workspace 决定的。
  2. 一个 accountId 最好只绑定一个明确 agent。 这样排查问题最方便。
  3. defaultAccount 只是默认账号，不等于默认 agent。 默认 agent 还是看 agents.list 里的 default: true 和 binding 命中情况。
  4. 账号层解决“谁来接消息”，workspace 层解决“接到后像谁”。

✅ 你现在这套 Discord 配置的优点

  • 多账号已经拆开
  • bot 名称语义清楚
  • 绑定关系简单，排查时不容易迷路
  • 一个账号对应一个 agent，结构很干净

📈 你以后最可能想加的两类东西

  • guild/channel 级 allowlist：决定 bot 在哪些服务器、哪些频道可以说话
  • per-agent skill allowlist：让不同 bot 真正拥有不同技能集

🧪 常见故障怎么反推

“这个 bot 不回消息”先查 channels.discord.enabled、token、账号是否在线、groupPolicy/guild allowlist。
“A bot 回出来像 B bot”先查 bindings 是否把 accountId 绑错 agent。
“多个 bot 回答能力都差不多”不是 bug，先查是不是根本没做 agents.list[].skills 分化。
“bot 说话风格不对”不要先看 token，先看对应 workspace 的 SOUL.md / USER.md / AGENTS.md。
🔐 不要把 Discord token、gateway token、app secret 发到聊天或文档网站里。这个站里只讲结构，不展示真实 secret。
