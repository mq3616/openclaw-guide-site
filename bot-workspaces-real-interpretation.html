你当前 Bot 工作区文件的真实解读 🧾

这一页不再只讲“AGENTS.md 一般干什么”，而是结合你当前 bot workspace 里真实写了什么，来解释这些 bot 的实际分工和气质。

📈 trading_report_bot，规则最像“纪律型交易执行官”

从它的 AGENTS.md 能看出几个非常明确的信号：

  • Primary domain = investment_finance，主域就是投资金融
  • Reply mode = calm_numeric，说明它偏冷静、偏数字化、偏结构化
  • 启动时强制读投资系统入口、持仓主表、收益统计总览
  • 明确规定“交易所截图盈亏不是权威成本基线”
  • 很多问题都要求先走特定流程，比如先查钱包、先做 buy judgment、先做 portfolio reconcile

这意味着：这个 bot 不是给你聊市场情绪的，它更像一套纪律化交易执行接口。它被设计成尽量靠系统事实源回答，而不是靠临场感觉。

🛰️ info_collection_bot，规则最像“情报采集与 staging 机器人”

它的 AGENTS.md 比 trading bot 更长，而且非常像一个工作流机器人：

  • Primary domain = information_collection
  • Secondary domain = knowledge_organization
  • 强调它不是绑在某一个业务系统模块上
  • 强调 source-based output、structured 整理、优先写 staging
  • 还有一整套 standing order，专门做 AI / 医疗 / 系统 intelligence collection

这意味着：它不是来给最后判断的，而是来做搜集、暂存、整理、digest、promotion candidate 的。更像前线侦察兵 + 情报整理器。

🌿 family_health_bot，优先级明显是“关系稳定 > 效率”

它的 AGENTS 很短，但信号很强：

  • Primary domain = family_life
  • Secondary domain = health_medical
  • Reply mode = high_eq_relationship
  • 明确写了：Relationship stability comes before efficiency
  • 明确写了：No hard diagnosis

这意味着：这个 bot 的第一目标不是“快答对”，而是“稳、温和、不造成关系或医疗风险”。这其实是非常像照护型 bot 的配置。

📚 library_growth_bot，核心不是聊知识，而是“组织知识”

它的 AGENTS 里最关键的不是学习这个词，而是这些规则：

  • Prefer structured notes over generic inspiration
  • Never fake having read a source
  • For library-domain answers, use glossary/card/wiki/query retrieval before raw source
  • If materially used the library, make a default writeback judgment

这意味着：它更像图书馆系统的运营者，而不是泛泛的读书 bot。它天然偏“先组织、后回答；先检索结构、后看原文”。

🧩 skills_master_bot，约束感非常强

它的 AGENTS 里最有代表性的几句是：

  • This bot is dedicated to library-skills work
  • Prefer minimal correct edits over broad refactors
  • Keep atomic skills and workflow skills strictly separated
  • Do not pollute library-skills with OB business content

这意味着：skills_master_bot 不是用来发散 brainstorm 的，它像一个有 repo hygiene 洁癖的技能工程 bot。它的价值是把 skill 仓、workflow、registry、边界守干净。

🪞 SOUL.md 和 USER.md 的真实作用，在你这套系统里是什么

你现在几个 bot 的 SOUL.md 很接近，说明它们共享一套“大人格基线”，例如：

  • 不要表演式热情
  • 先自己查，不要一上来问用户
  • 别越权，别乱代表用户

真正把 bot 拉开差距的，更多是：

  • AGENTS.md 里的岗位规则
  • USER.md 里的用户偏好和场景约束

比如 trading bot 的 USER 明确写了：先看实时市场、先做 trade-worthiness score；info bot 的 USER 明确写了：OB 是主记录点、OpenClaw 是代理层、用户学习内容和 AI 执行内容要分层存储。

💡 一个很关键的观察：你这套 bot 的差异，主要是“岗位规则差异”，不是“灵魂模板差异”。 这其实是成熟做法，因为共用一套基础人格，再用岗位规则去分工，系统会更稳。
