常见故障排查 🩺

这一页只做一件事，遇到问题时帮你尽快定位到该查哪一层。

1. Bot 不回消息

  1. 先查 Discord 账号 token / enabled
  2. 再查 groupPolicy / allowlist
  3. 再查 binding 是否把这个 accountId 路由到正确 agent
  4. 最后查 gateway 是否正常运行

2. Bot 回消息了，但像另一个 bot

  1. 先查 bindings[]
  2. 再查目标 agent 的 workspace
  3. 最后查该 workspace 里的 SOUL.md / USER.md / AGENTS.md

3. Bot 风格对了，但能力没分开

  1. 先确认是不是根本没配 agents.list[].skills
  2. 再查有没有独立 skills/ 目录
  3. 最后才看 skill 本身是否可用

4. Bot 太啰嗦 / 太吵 / 太爱插话

  1. 先查 USER.md
  2. 再查 SOUL.md
  3. 再查 AGENTS.md 和 HEARTBEAT.md
  4. 不要一上来就改模型

5. 改完一个 bot，别的 bot 也出问题

  1. 先回看你是不是改了公共层，比如 agents.defaults / shared docs
  2. 再查是不是改了共用 skill 池
  3. 最后确认是不是误改了 binding 或 defaultAccount
💡 排查顺序建议固定成一句话：先账号，后路由，再 workspace，最后技能和模型。
