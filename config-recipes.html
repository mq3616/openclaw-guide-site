常见配置修改实例 🧪

这一页专门讲“如果你真要改”，通常该改哪几处。不是泛泛的概念页，而是操作思路页。

场景 1，新增一个 Discord Bot

  1. 先在 channels.discord.accounts 增加一个新 accountId 和 token
  2. 在 agents.list[] 增加一个对应 agent，给它独立 workspace
  3. 在 bindings[] 增加一条，把这个 accountId 路由到该 agent
  4. 给新 workspace 准备 AGENTS.md / SOUL.md / USER.md
{
  agents: {
    list: [
      { id: "research_bot", workspace: ".../bot-workspaces/research_bot" }
    ]
  },
  bindings: [
    { agentId: "research_bot", match: { channel: "discord", accountId: "research_bot" } }
  ],
  channels: {
    discord: {
      accounts: {
        research_bot: { token: "..." }
      }
    }
  }
}

场景 2，把某个 bot 换到新的 workspace

如果你不满意某个 bot 当前人格，不一定要重建 bot 账号。很多时候只要改：

agents.list[].workspace

再把新 workspace 里的 AGENTS/SOUL/USER 配好就行。

⚠️ 这会明显改变 bot 的人格和记忆读取环境，所以改之前最好先备份原 workspace。

场景 3，让两个 bot 真正技能不同

不是只改 workspace，而是给 agent 配 skill allowlist：

{
  agents: {
    defaults: { skills: ["weather", "tmux"] },
    list: [
      { id: "trading_report_bot", skills: ["weather"] },
      { id: "skills_master_bot", skills: ["skill-creator", "clawhub", "tmux"] }
    ]
  }
}

这样才是能力层真正分流。

场景 4，某个 bot 回答太啰嗦

不要先改模型，也不要先改技能。更稳的顺序通常是：

  1. 查 bot workspace 的 USER.md
  2. 查 SOUL.md
  3. 查 AGENTS.md
  4. 最后才考虑 thinkingDefault 或模型

场景 5，某个 bot 在不该回的地方乱回

  1. 先查 channels.discord.groupPolicy
  2. 再查 guild/channel allowlist
  3. 再查 bot workspace 的 AGENTS / HEARTBEAT 里有没有鼓励它乱插话
💡 很多配置修改问题，真正稳定的原则都是：先查路由，再查人格，再查技能，最后才查模型。
