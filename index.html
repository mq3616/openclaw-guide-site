OpenClaw 常用配置文件精讲

这套站点不是照搬官方文档，而是围绕你当前机器上的实际结构，把 ~/.openclaw/openclaw.json、多 Bot 路由、Bot 工作区文件、Skills 个性化这些最容易混淆的部分拆开讲清楚。

目标很直接：看完之后，你能判断该去哪里改、改哪一层、为什么这样改更稳。

这套站最适合什么时候看

  • 想搞清楚一个 Discord Bot 最终为什么会接到某个 agent。
  • 想理解 openclaw.json 里常见 key 的真实作用。
  • 想分辨 Bot 的“人格差异”和“能力差异”分别来自哪里。
  • 想以后自己改 Bot 配置，而不是每次靠猜。

你当前这套系统的核心事实

  • 你现在是多账号、多 Bot 的接线方式，不同入口会路由到不同 agent。
  • 真正的行为差异，很多时候来自各自的 workspace 文件，而不只是模型名。
  • 如果不同 agent 没有单独收紧 skills，那它们的能力边界会比较接近。
  • 要做稳定分工，就得把路由层、人格层、技能层分开理解。

推荐阅读顺序

  1. 先看 配置地图，建立整体文件层级感。
  2. 再看 openclaw.json 详解，理解总控台。
  3. 然后看 Bot / Agent / Binding，搞懂路由。
  4. 接着看 Bot 工作区文件，理解人格和工作规则。
  5. 最后看 Discord 配置重点 和 Skills 与个性化。

看完之后你应该获得什么

  • 知道哪些配置该去 ~/.openclaw 改，哪些该去 Bot workspace 改。
  • 知道 Bot 账号、agent、binding 不是一回事。
  • 知道人格差异和能力差异分别由什么决定。
  • 知道以后如果要把不同 Bot 真正做出分工，该从哪一层开始收紧。
