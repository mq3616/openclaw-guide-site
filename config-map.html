配置地图，先建立文件层级感

理解 OpenClaw 最容易混乱的地方，就是把 ~/.openclaw、主 workspace、bot workspace 这三层混在一起。你先把它们分开，后面很多事就顺了。

第一层，系统层

~/.openclaw/
├─ openclaw.json
├─ credentials/
├─ memory/*.sqlite
├─ cron/
├─ agents/
└─ ...

这一层负责系统总控，包含总配置、凭据、会话、内部运行状态。最重要的是 openclaw.json。

第二层，主工作区

~/.openclaw/workspace/
├─ AGENTS.md
├─ SOUL.md
├─ USER.md
├─ MEMORY.md
├─ memory/
└─ runtime-adapter/

这一层是主 agent 的“家”。文件型记忆、主人格、主用户偏好都在这里。

第三层，bot 独立工作区

runtime-adapter/bot-workspaces/
├─ trading_report_bot/
├─ family_health_bot/
├─ library_growth_bot/
├─ info_collection_bot/
└─ skills_master_bot/

每个 bot 都像独立员工工位，有自己的 AGENTS/SOUL/USER/TOOLS/HEARTBEAT/memory。

最实用的判断法

想看什么该去哪
机器人账号接到哪个脑子~/.openclaw/openclaw.json
这个 bot 说话风格bot-workspaces/<bot>/SOUL.md
这个 bot 的工作规则bot-workspaces/<bot>/AGENTS.md
这个 bot 怎么服务你bot-workspaces/<bot>/USER.md
定时醒来时做什么bot-workspaces/<bot>/HEARTBEAT.md
技能允许范围先看 openclaw.json 的 agents.*.skills，再看 skill 根目录
不要把 ~/.openclaw 和 workspace 混为一谈。前者更像系统后台，后者更像“给 agent 读的生活和工作资料”。
