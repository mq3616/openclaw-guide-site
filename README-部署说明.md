# OpenClaw 精讲站部署说明

这个项目是纯静态网站，目录里只有 `html`、`css`、`js` 文件，没有后端依赖，所以最适合发布到静态托管平台。

## 最推荐的发布方式

### 方案一：Cloudflare Pages

适合想要：

- 免费上线
- 自动获得公网网址
- 手机直接打开
- 后续可绑定自己的域名
- 需要基础访问统计

操作步骤：

1. 把整个 `OpenClaw精讲站` 文件夹上传到一个 GitHub 仓库。
2. 登录 Cloudflare，进入 `Workers & Pages`。
3. 选择 `Create application`。
4. 选择 `Pages`。
5. 连接 GitHub 仓库。
6. 选择这个项目仓库。
7. 构建设置里保持静态站默认即可：
   - Build command 留空
   - Output directory 填 `.` 
8. 点击部署。

部署完成后，Cloudflare 会给你一个默认网址，通常类似：

`https://你的项目名.pages.dev`

这个网址就是你可以发到手机里直接访问的公网地址。

### 方案二：Netlify

适合想快速测试，不一定先建仓库。

操作步骤：

1. 登录 Netlify。
2. 进入 `Add new site`。
3. 直接把整个 `OpenClaw精讲站` 文件夹拖进去。
4. 等待上传和发布完成。

发布后会得到一个默认网址，通常类似：

`https://项目名.netlify.app`

### 方案三：GitHub Pages

适合你本来就准备把这个站放进 GitHub 仓库长期维护。

操作步骤：

1. 新建 GitHub 仓库并上传全部文件。
2. 打开仓库的 `Settings`。
3. 进入 `Pages`。
4. 选择从当前分支发布。
5. 根目录选择 `/ (root)`。
6. 保存并等待生成网址。

发布后会得到类似：

`https://你的用户名.github.io/仓库名/`

## 网址从哪里来

你会遇到两类网址：

### 平台默认网址

这是发布平台自动送你的，最省事，不需要自己买域名。

示例：

- `你的项目.pages.dev`
- `你的项目.netlify.app`
- `你的用户名.github.io/仓库名`

这类网址已经可以直接发给别人，也可以在手机浏览器里直接访问。

### 自定义域名

如果你想让网址更正式，比如：

`openclaw.yourdomain.com`

那你需要先去买一个域名，比如在阿里云、腾讯云、Namecheap、Cloudflare Registrar 上购买，然后到托管平台里绑定。

绑定的本质是：

1. 你有一个自己的域名
2. 在托管平台里添加这个域名
3. 按平台提示修改 DNS 解析
4. 等证书签发完成

之后手机访问的就是你自己的网址。

## 怎么收集访问数据

如果你只是想知道：

- 有多少人访问
- 哪些页面最常被打开
- 手机访问多还是电脑访问多

最简单的是以下几种：

### Cloudflare Web Analytics

适合 Cloudflare Pages 用户。

优点：

- 接入简单
- 页面性能影响小
- 不用你自己写后端

### Netlify Analytics

适合 Netlify 用户。

优点：

- 和站点托管集成紧
- 不用单独搭统计系统

### Google Analytics

适合以后你想看更详细的来源、设备、停留时长。

优点：

- 功能完整
- 行业通用

缺点：

- 配置比前两种稍复杂

## 手机上怎么访问

正式发布后，你只要在手机浏览器里打开平台提供的网址或你绑定的域名即可。

如果你只是临时预览，也可以在电脑本机开静态服务，再用同一 Wi-Fi 下的局域网 IP 访问，但这种方式：

- 电脑必须开着
- 只能在同一个局域网下访问
- 不算正式发布

## 当前项目发布前建议

这个项目已经具备静态发布条件。

发布前建议确认：

1. 首页内容显示正常，没有乱码。
2. 手机宽度下导航和卡片布局正常。
3. 所有页面链接都能打开。
4. 不要把真实 token、secret、认证信息写进页面里。

## 我推荐你的实际路线

如果你想省事，直接走：

1. GitHub 仓库
2. Cloudflare Pages
3. 先用 `pages.dev` 默认网址
4. 后面需要更正式时再绑定自己的域名
5. 打开 Cloudflare Web Analytics 做访问统计

这样成本最低，后续也最好维护。
