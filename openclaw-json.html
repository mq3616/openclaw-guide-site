~/.openclaw/openclaw.json 怎么看 🧩

你当前文件的 top-level key 是：meta、wizard、browser、auth、agents、bindings、tools、commands、session、channels、gateway、plugins。这一页会告诉你，这些 key 分别是“系统记录”“日常常改”“多 bot 必懂”里的哪一类。

🪟 先记一句话

openclaw.json = 总控台。 它决定系统里有哪些 agent、哪些 channel 账号、这些账号怎么路由、系统工具和 gateway 怎么开。你可以把它理解成 OpenClaw 的“主配电箱”。

🗂️ top-level key 逐个讲

meta系统写回的元信息，比如最近一次被哪个 OpenClaw 版本触碰。通常只看，不手改。
实用判断：看到它别紧张，它不决定 bot 行为。
wizard配置向导的运行痕迹。告诉你上次什么时候跑过 configure/setup。也是偏系统记录。
实用判断：适合排查“这台机器最近是不是跑过向导”。
browser内置浏览器/自动化浏览能力的配置，比如 Chrome 路径、是否无沙箱、额外参数。
你当前：启用了系统 Chrome，走 headless。
auth模型提供商认证资料的引用层。比如 openai-codex 的 oauth profile、openai 的 api key profile。
注意：这里是认证组织层，不是 bot 人格层。
agents最重要之一。 定义有哪些 agent，以及默认继承什么配置。
一句话：这里定义“系统里有哪些脑子”。
bindings最重要之一。 把进入的消息按 channel/accountId/peer 等条件路由到具体 agent。
一句话：这里定义“消息最终交给谁”。
tools工具总控，比如 web search、elevated tools、一些系统级开关。
一句话：这里定义“系统普遍允许用哪些工具能力”。
commands命令系统的行为，比如 native command 是否自动启用、restart 等。
一句话：这里定义“命令系统怎么表现”。
session会话切分策略，比如 DM 是按什么粒度合并或拆分。
一句话：这里定义“上下文怎么分桶”。
channels所有对外渠道配置。你这里重点是 channels.discord 和 channels.feishu。
一句话：这里定义“外部入口有哪些、怎么接”。
gateway本地网关本身怎么跑，比如端口、bind、认证 token、tailscale、nodes 限制。
一句话：这里定义“OpenClaw 服务本身怎么启动和暴露”。
plugins插件开关和插件层配置，比如 duckduckgo / feishu / openai 等。
一句话：这里定义“有哪些插件模块打开了”。

🤖 agents 深入看

agents.defaults所有 agent 的公共默认配置。你这里包含：model.primary、fallbacks、workspace、thinkingDefault、memorySearch。
理解法：它像“全公司通用制度”。
agents.defaults.model.primary默认主模型。你的系统默认是 openai-codex/gpt-5.4。
影响：没单独指定模型的 agent，会优先用它。
agents.defaults.model.fallbacks主模型不可用时的后备模型列表。
影响：保证系统在主模型异常时不至于完全失能。
agents.defaults.models为特定模型挂额外配置的入口。你这里目前只是声明了 openai-codex/gpt-5.4。
进阶用途：后面如果你想对某模型挂额外设置，可以从这里走。
agents.defaults.workspace默认工作区路径。没单独写 workspace 的 agent 会继承它。
重要性：workspace 决定人格文件、memory、local notes 从哪读。
agents.defaults.thinkingDefault默认思考强度。你的公共默认是 medium。
理解法：这不是“聪明程度”，更接近“愿意花多少脑力”。
agents.defaults.memorySearch.local/provider默认记忆检索方式。你这里是本地 memory search。
影响：决定它回忆旧内容时优先从哪里查。
agents.list[]每一个实际 agent 的定义。你这里有 main、trading_report_bot、family_health_bot、library_growth_bot、info_collection_bot、skills_master_bot。
理解法：这是“员工名单”。
agents.list[].idagent 的唯一标识。binding 最终就是路由到这个 id。
注意：这是路由世界里的真名。
agents.list[].workspace该 agent 的工作区目录。非常关键，因为人格文件、用户偏好、memory 都在这里。
影响：两个人设差异，很多时候不是因为 skill，不是因为模型，而是因为 workspace 不同。
agents.list[].thinkingDefault该 agent 覆盖默认思考强度。你这里 trading/family/library 是 low，skills_master 是 high。
实战理解：技能大师 bot 会更愿意深想，交易/家庭/图书馆 bot 更偏轻量响应。
agents.list[].identity这个 agent 的展示身份，比如 name、emoji、avatar、theme。
影响：主要是展示和角色识别，不是完整人格本体。
agents.list[].default是否默认 agent。你这里 main 是默认。
常见误区：默认 agent 不等于默认 Discord 账号。

⚙️ 你当前文件里其他常用 key，再往下一层

browser.enabled是否启用浏览器能力。
browser.executablePath浏览器可执行文件路径。你这里指向系统 Chrome。
browser.noSandbox浏览器启动时是否关闭 Chromium sandbox。某些 Linux/WSL 环境经常会这样配。
browser.extraArgs[]额外启动参数，比如 headless、禁用 GPU、跳过首次启动检查。
auth.profiles模型认证档案。一个 profile 对应一个 provider 的登录或 key 入口。
tools.profile工具 profile。你这里是 coding，会影响工具侧默认组合。
tools.web.search.enabled/providerWeb 搜索能力和默认 provider。你这里是 DuckDuckGo。
tools.elevated.enabled是否允许 elevated 工具能力。
tools.elevated.allowFrom哪些来源可以直接触发 elevated。你这里是 webchat / feishu 的特定来源。
commands.native是否自动启用 native commands。
commands.nativeSkills是否自动启用 native skill commands。
commands.restart是否允许 restart 相关命令。
commands.ownerDisplay命令输出里 owner 的展示方式。
session.dmScopeDM 会话怎么切。你这里是 per-channel-peer，表示按渠道和对端粒度分会话。
理解法：这影响“同一个人从同一个渠道发来的消息，是否会进同一条上下文”。
gateway.portGateway 监听端口。你这里是 18789。
gateway.mode网关运行模式。你这里是本地 local。
gateway.bind监听绑定范围。你这里是 loopback，说明默认只本机可连。
gateway.auth.mode/tokenGateway 自己的认证方式和 token。也是敏感项。
注意：这和 Discord token 不是一回事。
gateway.tailscale是否通过 tailscale 暴露网关。
gateway.nodes.denyCommands对节点侧命令做 denylist。你这里禁掉了相机、录屏、短信等命令。
理解法：这是节点安全边界的一部分。
plugins.entries.*.enabled插件是否启用。你当前启用了 duckduckgo、feishu、openai 等。

🔍 怎么看一个 key 值得不值得你平时关心

  • 高频关注：agents、bindings、channels.discord、tools
  • 中频关注：browser、gateway、session
  • 低频关注：meta、wizard、大部分 auth.profiles 细节

🧠 真正常见的实战问题

  • 某个 bot 风格不对，多半先查 workspace 而不是查 token。
  • 某个 bot 收到消息却像另一个 bot，先查 bindings。
  • 多个 bot 能力看起来一样，先查有没有 agents.list[].skills。
  • 网页或桌面打不开，不先看 bot 配置，先看 gateway / 本地文件路径。

🎯 你当前最值得用好的三块

  1. agents：决定“有哪几个脑子”。
  2. bindings：决定“哪个 bot 账号接到哪个脑子”。
  3. channels.discord.accounts：决定“接了哪些 Discord 账号”。
🔐 这个文件里会包含 token / secret / gateway token / bot token。学习结构可以，但不要把原文件完整发给外部。
