为什么你现在是“多 bot，共用 skill 池” 🛠️

你现在虽然已经是多 agent、多 bot 结构，但并没有给 agent 单独配置 agents.defaults.skills 或 agents.list[].skills，也没有在各 bot workspace 下放独立 skills/。所以当前更像是：多 bot 用不同人格，但能力池基本共用。

📚 技能加载的几个来源

<workspace>/skills某个 workspace 自己的 skills，优先级最高。
适合：这个 bot 独占的本地技能。
<workspace>/.agents/skills该 workspace 的 agent skill 目录。
适合：比 workspace 通用一点，但仍偏 agent 内部。
~/.agents/skills个人 agent skills。
~/.openclaw/skills受管的本地 skills。
bundled skillsOpenClaw 自带技能。
skills.load.extraDirs额外扫描目录，优先级最低。

🎛️ 控制“谁能看到哪些 skill”的关键 key

skills.allowBundled只限制 bundled skills，控制哪些自带技能可见。
skills.entries.<name>某个 skill 的启停、apiKey、env 等。
agents.defaults.skills所有 agent 共用的默认 skill allowlist。
理解法：全公司通用工具包。
agents.list[].skills某个 agent 的最终 skill allowlist。写了就会替换 defaults，而不是叠加。
理解法：某岗位专属工具包。
💡 这也是为什么我之前说，个性化 skill 不是按 Discord bot 账号本身做，而是按 agent 做。bot 账号只是入口，真正接技能的是 agent。

📌 你当前状态

  • 没有看到 agents.defaults.skills
  • 没有看到 agents.list[].skills
  • 各 bot workspace 下也没有独立 skills/

所以当前结论是：技能基本共用，差异主要来自 workspace 人格文件。

🧪 为什么这件事很容易误判

  • 你看到 bot 名字不同，以为它们自然就有不同能力。
  • 你看到 workspace 不同，以为 skill 也已经分开了。
  • 你看到回答风格差很多，于是误以为技能也完全不同。

其实不一定。风格差异 ≠ 能力差异。风格大多来自 SOUL/USER/AGENTS，能力是否分开要看 skill allowlist 和 skill 目录。

🚀 以后如果你想把 bot 真正做出岗位分工

{
  agents: {
    defaults: {
      skills: ["weather", "tmux"]
    },
    list: [
      { id: "trading_report_bot", skills: ["weather"] },
      { id: "skills_master_bot", skills: ["skill-creator", "clawhub", "tmux"] }
    ]
  }
}

这样才叫真正意义上的“不同 bot 有不同 skill 套餐”。

⚠️ 但现实里，很多 bot 用不好，并不是 skill 不够，而是角色边界和 workspace 文件没写清。所以通常建议：先收人设，再收技能。
