Bot 工作区文件，才是真正的“人格层” 🧠

你的各 bot 目录都在 ~/.openclaw/workspace/runtime-adapter/bot-workspaces/<bot>/。这一层决定 bot 的风格、边界、工作流程、连续性。很多人一开始会把“bot token”“agent id”“人格文件”混成一团，其实真正决定 bot 像谁、怎么做事的，大多都在这里。

📁 每个 bot 目录里常见文件

AGENTS.md
SOUL.md
USER.md
TOOLS.md
HEARTBEAT.md
IDENTITY.md
memory/
.openclaw/

你可以把一个 bot workspace 想成一个岗位工位。桌上摆的每份文件，都在告诉这个 bot “你是谁、你服务谁、你平时怎么做事、你醒来要干什么”。

🪪 这些文件分别控制什么

AGENTS.md工作制度文件。 写启动规则、记忆规则、边界、heartbeat 用法、长期工作方式。最像“岗位操作手册”。
典型用途：规定启动时读哪些文件、哪些场景必须先问用户、什么能自动做、什么不能自动做。
SOUL.md人格文件。 决定口吻、性格、风格、边界感。最像“气质说明书”。
典型用途：让交易 bot 更像分析助手，让家庭 bot 更像温和照护型，而不是所有 bot 都说一样的话。
USER.md服务对象偏好文件。 写用户怎么称呼、喜欢什么风格、哪些动作要先确认。最像“客户偏好卡”。
典型用途：规定回复要短、不要贴整篇文档、敏感操作先确认。
TOOLS.md本地环境备忘录。 记录脚本路径、设备名、目录约定、环境注记。不会决定工具可用性，但会影响 bot 怎么用工具。
典型用途：写本地脚本在哪、某个目录代表什么、某个设备别名是什么。
HEARTBEAT.md被定期唤醒时该做什么。 比如检查收件箱、跑日报、看项目推进、决定是否要主动打扰你。
典型用途：让情报 bot 定时扫描，让健康 bot 定时关注提醒，而不是每次 wake 都只回一句 OK。
IDENTITY.md身份卡。 结构化写名字、emoji、vibe、avatar 等。
典型用途：统一 bot 展示身份，方便识别。
memory/这个 bot 的文件型记忆区。 放 daily memory、项目沉淀、阶段快照等。
典型用途：让 bot 下次醒来还能知道最近发生了什么。
.openclaw/bot 工作区下的局部 OpenClaw 内部目录，偏运行辅助，不是最主要的人类编辑入口。

🎭 如果你想改 bot 的“人设”

  • 先看 SOUL.md
  • 再看 USER.md
  • 最后看 AGENTS.md

顺序很重要。很多时候不是 skill 不够，而是人格、边界、偏好没有写清。

一个经验：如果某个 bot 总是“会做事但你不喜欢它的方式”，那通常不是工具问题，是这三份文件没收好。

⚙️ 如果你想改 bot 的“做事方式”

  • 工作规则改 AGENTS.md
  • 主动性改 HEARTBEAT.md
  • 工具环境注记改 TOOLS.md
  • 连续性和沉淀看 memory/

一个经验：如果某个 bot 太吵、太勤、太爱多说话，先查 heartbeat 和 AGENTS，不要上来就怀疑模型。

🧪 用实际问题反推该改哪份文件

“这个 bot 回答太啰嗦”先查 USER.md 和 SOUL.md
“这个 bot 总在不该说的时候插话”先查 AGENTS.md 和 HEARTBEAT.md
“这个 bot 不记得上次说到哪”先查 memory/ 和启动读取规则
“这个 bot 用错目录 / 脚本 / 路径”先查 TOOLS.md
“这个 bot 风格不像它这个岗位”先查 SOUL.md
💡 一个很实用的经验：skill 决定“会不会做”，workspace 文件决定“怎么做”。 如果 bot 用着不顺，先改 workspace 文件，往往比先折腾 skill 更见效。
